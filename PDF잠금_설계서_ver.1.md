# 업무매뉴얼 PDF 잠금 설계서 ver.1
교통외근 통합보조 v9.2 기준 · 2026.6 · 구현 보류 상태 (설계 확정용)

## 1. 목적
index.html 내장 업무매뉴얼 PDF(base64)가 파일 유출 시 그대로 열람되는 문제를 차단한다.
화면 가림이 아니라 **데이터 자체 암호화**로, 소스 보기·파일 분석으로도 원본을 복원할 수 없게 한다.

## 2. 방식 — 주관식 정답 = 암호화 키 (AES-GCM)
- 매뉴얼 PDF 페이지 데이터(현재 PDF_PAGES 배열)를 **AES-256-GCM**으로 암호화하여 내장.
- 열람 시 주관식 질문 표시 → 입력한 정답에서 **PBKDF2(SHA-256, 60만회 반복, 고정 salt)**로 키 유도 → 복호화.
- 정답이 맞으면: 복호화 성공 → 갤러리 표시. 틀리면: 복호화 자체가 실패 (정답 비교 로직 없음 → 소스에 정답 흔적 0).
- 브라우저 내장 WebCrypto만 사용. **외부 통신 0건 유지, 단일 파일 유지.**

## 3. 동작 흐름
1. 상단 [📄 업무매뉴얼] 터치
2. 질문 모달: "OOO은?" (주관식 1문항, 필요 시 2문항 조합)
3. 정답 입력 → 키 유도 → 복호화 → 갤러리 오픈
4. 성공 시 **세션 동안 재질문 없음** (sessionStorage에 유도키 임시 보관, 앱 종료 시 소멸)
5. 5회 연속 실패 시 10분 입력 잠금 (무차별 대입 지연)

## 4. 정답 문구 정책 (형님 지정 — 미정)
요건:
- 외부인이 검색·추측 불가능할 것 (관서명·직급·공개 전화번호 금지)
- 내부 직원은 즉답 가능할 것 (교양·전파사항으로 공지 가능한 것)
- 한글 6자 이상 권장. 띄어쓰기·대소문자 처리 규칙 고정 (전부 제거 후 비교)
- 예시 유형: 내부 행정전화 뒷자리+약속 단어 조합, 부서 내 공지 문구 등

문구 변경 시: 새 문구로 재암호화한 배포본을 다시 생성해야 함 (빌드 스크립트 1회 실행).

## 5. 한계 (정직한 고지)
- 정답을 아는 내부자가 파일+정답을 함께 유출하면 무력 — 이는 어떤 기술로도 차단 불가, 복무규정 영역.
- 정답이 짧거나 추측 가능하면 PBKDF2 반복으로 지연돼도 결국 뚫림 → 문구 품질이 보안의 전부.
- 화면 캡처·촬영은 차단 대상 아님.

## 6. 구현 시 작업 항목 (승인 후)
1. 빌드 스크립트: PDF_PAGES → 암호문 변환 (Python, 1회 실행)
2. index.html: 복호화 모듈 + 질문 모달 + 실패 잠금 (약 60줄)
3. 버전 v9.3 승격 + 설정 업데이트 기록 추가
4. 검증: 정답/오답/5회 실패/세션 유지/오프라인 동작 테스트
